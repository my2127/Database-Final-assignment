## 2. プロジェクト概要
本プロジェクトは、家庭内での食材廃棄（フードロス）を削減することを目的としています。インターネット技術の発展が生活に与える影響を背景に、冷蔵庫内の在庫状況をリアルタイムに可視化することで、二重買いの防止や計画的な消費をサポートします。

### 主な機能
* **食材のCRUD操作**: 在庫の登録、一覧表示、削除を網羅。
* **高度な分類機能**: 12種類のカテゴリー（野菜、肉、魚、果物、乳製品、冷凍食品、飲み物、惣菜、お菓子、加工食品、調味料、その他）による管理。
* **視覚的なアラート機能**: 賞味期限切れまでの日数を自動計算。期限切れは赤、3日以内はオレンジに背景色を変化させ、直感的な把握を可能にします。
* **フィルタリングと検索**: 食材名によるキーワード検索、およびカテゴリー別の絞り込み。

## 3. システム構成 (Architecture)
保守性と拡張性を考慮した **Web 3-Layer Architecture** を採用しています。



| 層 (Layer) | 使用技術 | 役割 |
| :--- | :--- | :--- |
| **Presentation** | HTML5, CSS3, Jinja2 | ブラウザへの画面表示・動的なUI構築 |
| **Application** | Python 3.9, Flask | ビジネスロジックの処理、DB連携 |
| **Data** | PostgreSQL 15 | 食材データの永続化・管理 |

---

## 4. インフラ構成 (Infrastructure)
**Docker / Docker Compose** を活用し、OS環境に依存しない再現性の高い開発・実行環境を構築しています。

* **コンテナ構成**:
    * `app`: Flaskサーバー（Python実行環境）
    * `db`: PostgreSQLデータベースサーバー
* **非機能要件**:
    * **RPO/RTO**: 24時間/1時間の目標を定義し、障害からの迅速な復旧を考慮した設計。

---

リレーショナルデータベース（RDB）による構造化データの管理を行っています。
